1
Create a GitHub classic token
⏱ ~2 min
1.1
Open the token creation page
Go directly to this URL — it opens the New Personal Access Token form:
https://github.com/settings/tokens/new
⚠
GitHub will ask you to confirm your password before showing this page. That is normal — it is their sudo-mode security check. Enter your GitHub password when prompted.
1.2
Fill in the token details
On the New Personal Access Token page you will see these fields:
github.com/settings/tokens/new
Note (token name)
Expiration
ℹ
Set the Note to vector-pro-deploy so you can identify this token later. Expiration can be 90 days or No expiration.
1.3
Tick exactly two scope checkboxes
Scroll down to the Select scopes section. You need repo and workflow — nothing else is required.
repo
Full control of private repos (includes read/write)
admin:repo_hook
Not needed — skip
workflow
Allows pushing .github/workflows/ files — required for GitHub Actions deploy
everything else
Leave unchecked
⚠
Must be a classic token — NOT a fine-grained PAT. Fine-grained tokens cannot write .github/workflows/ files, which is exactly what the deploy needs. The page you want is: Tokens (classic) in the left sidebar, not Fine-grained tokens.
1.4
Click "Generate token" and copy it immediately
Scroll to the bottom and click Generate token. GitHub shows the token only once.
✓ Token created: ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx ← Copy this now. It disappears when you leave the page. Paste it into a text note temporarily.
💡
Paste the token into a temporary note or keep this tab open. You will need to paste it into VECTOR PRO in Phase 3. Never commit it to a repo or share it.

3
Deploy from VECTOR PRO
⏱ ~2 min
3.1
Open VECTOR PRO locally
Open a Terminal, navigate to the vector-pro folder, and start a local server:
# Navigate to your vector-pro folder cd ~/Downloads/vector-pro # Start local server python3 -m http.server 8080 # Then open in Chrome or Safari: http://localhost:8080
ℹ
You only need the local server to access the GitHub deploy panel — no camera needed for this step. The deploy works entirely via the GitHub API in your browser.
3.2
Scroll to "GitHub Deploy" in the right panel
In VECTOR PRO, scroll the right-hand side panel all the way down until you see the GitHub Deploy section (second from the bottom, above Drone Bridge).
GitHub Deploy not configured
GitHub Username
Repository name
Classic token (repo + workflow scope)
3.3
Fill in the three fields
Enter all three values exactly as shown:
GitHub Username digitalyticsagency Repository name vector-pro Classic token ghp_xxxx... ← paste from Step 1.4
⚠
The token field is a password input — it will show dots. That is intentional. The token is sent directly to the GitHub API from your browser and is never stored or transmitted anywhere else.
3.4
Tap "Init Repo" — watch the status
Click Init Repo. The app will push five files to GitHub and enable Pages. Watch the status box below the buttons:
GitHub Deploy live ✓
Checking repo… Repo exists → pushing files… ✓ index.html pushed (80KB) ✓ landing.html pushed (58KB) ✓ .github/workflows/deploy.yml pushed ✓ bridge/drone_bridge.py pushed ✓ README.md pushed Enabling GitHub Pages… ✓ Pages enabled on main branch → https://digitalyticsagency.github.io/vector-pro/ (live in ~60 seconds)
ℹ
If it says 404 Not Found on the repo check, the repo does not exist yet — go back to Phase 2 and create it first, then return here.
3.5
GitHub Actions builds and deploys automatically
The pushed .github/workflows/deploy.yml triggers a GitHub Actions run that publishes the site. You can watch it at:
https://github.com/digitalyticsagency/vector-pro/actions
Typical Actions timeline: 0s Push received by GitHub ~5s Actions workflow queued ~20s Pages build starts ~40s Pages artifact uploaded ~60s Pages deployment complete ✓ ~65s URL live → https://digitalyticsagency.github.io/vector-pro/
4
iPhone setup — fullscreen + rear camera
⏱ ~1 min
4.1
Open in Safari (not Chrome)
On your iPhone, open Safari (not Chrome — only Safari supports Add to Home Screen with fullscreen camera). Navigate to your URL:
https://digitalyticsagency.github.io/vector-pro/
⚠
Camera access requires HTTPS — the GitHub Pages URL provides this automatically. The app will not work if you open it as a file:// or http:// URL on your iPhone.
4.2
Add to Home Screen for fullscreen mode
In Safari, tap the Share button (box with arrow pointing up) → scroll down and tap "Add to Home Screen" → tap Add. Now open VECTOR PRO from your Home Screen.
Safari → Share button (□↑) → "Add to Home Screen" → Name: "VECTOR PRO" → Add Result: fullscreen app, no Safari chrome, rear camera active
💡
When opened from Home Screen, VECTOR PRO runs in standalone mode (no address bar). The app detects facingMode: environment and automatically switches to rear camera + turns off mirror mode.
4.3
Select maximum resolution
In the Sensor panel (right side), set Resolution to the highest available for your iPhone:
iPhone 15 Pro / Pro Max: 4K UHD — 3840×2160 iPhone 14 Pro / 15: FHD — 1920×1080 (safe choice) iPhone 12/13 standard: HD — 1280×720 (if FHD stalls) If the camera freezes at a high resolution, drop one level down.
4.4
Check optical zoom availability
iPhone Pro models (12 Pro and later) expose optical zoom via the getCapabilities() API. If your device supports it, a Zoom slider will automatically appear in the Sensor section after the camera starts. Slide it to use 2× or 3× optical zoom for distant objects.
ℹ
The zoom slider only appears when the browser reports zoom capability. If it does not appear, your camera hardware or iOS version does not expose this via the web API — use the native Camera app for optical zoom and crop the view instead.

5.3
Troubleshooting common issues
PROBLEM: Camera blocked / not starting FIX: Must use HTTPS URL. file:// and http:// block camera. Open https://digitalyticsagency.github.io/vector-pro/ PROBLEM: "Init Repo" returns 404 FIX: Repo does not exist yet. Do Phase 2 first, then retry. PROBLEM: "Init Repo" returns 409 Conflict on SHA FIX: File already exists, app will fetch its SHA and update it. Wait 5s and tap Deploy Update instead. PROBLEM: Token rejected (401 Unauthorized) FIX: Wrong token type. Must be CLASSIC token, not fine-grained. Go to github.com/settings/tokens and check the token type. PROBLEM: Actions workflow fails (Error: Resource not accessible) FIX: Token missing workflow scope. Create a new token with both repo + workflow checked. PROBLEM: URL shows 404 after deploy FIX: Wait 60-90 seconds. Pages needs time to build. Check Actions tab for build status. PROBLEM: YOLOv8n fails to load (black screen after start) FIX: Model (~6.7MB) needs network. On slow connections it times out and falls back to MediaPipe automatically. Check the ENGINE stat in the top bar — MP-GPU is fine. PROBLEM: iPhone rear camera not activating FIX: Must open from Home Screen (Add to Home Screen first). In Safari without Home Screen, it defaults to front camera.
