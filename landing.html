Core technology

Every layer
engineered.

Five layers of computer vision, from raw pixels to drone commands, all running in a single browser tab.

⬡
YOLOv8n Detection
Quantized int8 ONNX model (~6.7MB) runs on WebGL GPU. 80 COCO classes. Falls back to MediaPipe EfficientDet, then TFJS COCO-SSD.
L1 · Perception
⇄
ByteTrack Tracking
Two-round association: high-confidence detections first, then low-confidence against unmatched tracks. Alpha-beta filter for smooth position estimates.
L2 · Tracking
🔒
Persistent Re-ID
Every track gets a 20-dim HSV appearance signature. Departed targets are held in a gallery — returning objects reclaim their original ID. Drone lock survives the gap.
Re-Identification
⚡
Speed Estimation
Pixel velocity projected through pinhole geometry gives per-object speed in km/h. 8-frame EMA removes jitter. Displayed on every label.
L3 · Analytics
⊕
Counting Lines & Zones
Draw virtual tripwires and alert regions directly on the video. Segment-intersection crossing detection counts IN/OUT per line. Zone occupancy fires instantly.
L3 · Analytics
🚁
Drone Guidance
Pinhole range estimation + P-controller outputs YAW, PITCH, FWD commands streamed over WebSocket. Bridge script relays to MAVLink for ArduPilot/PX4.
L4 · Guidance

Understanding.

The same analytics toolkit used by computer vision engineers, built directly into the interface.

〰
Counting lines
Click ╱ LINE, then tap two points on the video. A glowing tripwire appears. Every object that crosses it increments IN or OUT — the direction is detected automatically from the crossing vector.
▭
Alert zones
Drag ▭ ZONE to draw a region. The zone glows and shows live occupancy count when any tracked object enters. Zero false positives — the count is based on the track centroid.
🌡
Density heatmap
Every tracked centroid accumulates to a Float32 grid. Rendered with a jet colormap (blue → red) scaled to the viewport. Decays over time so you see where objects have been, not just where they are.

MacBook + iPhone

One file.
Every device.

No installation. No app store. A single HTML file that maximises whatever camera hardware you have.

01
MacBook — localhost
python3 -m http.server 8080 then open Chrome. Uses the highest resolution the FaceTime or external camera supports, up to 4K.
02
iPhone — GitHub Pages
Open the deployed URL in Safari. Add to Home Screen for fullscreen. Rear camera activates automatically, mirroring turns off. Optical zoom exposed if the device supports it.
03
Drone bridge
Connect the app to ws://localhost:8765 and run the Python bridge. Locked target telemetry streams at ~30Hz as MAVLink body-rate commands.
Architecture

Five layers, one file.

L1
Perception
YOLOv8n ONNX → MediaPipe → TFJS. Highest-performing available engine selected automatically.
L2
Tracking
ByteTrack 2-round IoU association + alpha-beta filter. Appearance Re-ID gallery for persistent IDs.
L3
Analytics
Speed in km/h · Counting lines · Zone alerts · Jet heatmap. All rendered on a second canvas layer.
L4
Guidance
Pinhole range + P-controller → WebSocket → Python bridge → MAVLink. Failsafe decays on silence.
Built-in GitHub deploy

Live on the web
in 60 seconds.

The app pushes itself to GitHub Pages. No terminal required.

01
Create a classic GitHub token
Settings → Developer settings → Personal access tokens → Tokens (classic). Select repo and workflow scopes.
02
Fill in the GitHub panel
Open VECTOR PRO → scroll to GitHub Deploy in the right panel. Enter your username, repo name, and token.
03
Tap Init Repo
The app creates the repo, pushes index.html + deploy workflow, and enables GitHub Pages automatically.
https://your-username.github.io/vector-pro/
